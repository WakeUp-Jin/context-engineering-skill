# 上下文问题与解决方案

上下文失控有四种典型情况，每种都可能导致 Agent 以意识不到的方式失败。

## 1. 上下文污染

**定义**：幻觉或错误进入上下文，并被反复引用。

**表现**：Agent 发展出无意义的策略，重复行为，追求无法实现的目标。

**典型场景**：

- **用户记忆污染**：用户情绪化表达被误解为持久偏好
  - 用户说"这节课烦死了，根本没用" → 系统整理为"用户认为《课程C》没有价值"
  - 后续所有涉及课程 C 的回答都被这条错误记忆影响

- **工具调用错误污染**：工具错误信息不准确，导致模型在错误方向上反复重试
  - 工具返回模糊的错误信息，LLM 无法理解真正原因
  - 反复重试只会积累更多错误上下文

**解决方案**：

```
1. 工具错误重试：
   - 引入最大重试次数
   - 引入 LLM 反馈机制：重试失败后让模型将错误反馈给用户
   - 工具错误信息要准确反映真正原因

2. 记忆模块：
   - 设计记忆更新和精简机制
   - 区分情绪表达和事实陈述
   - 记忆要有时效性和置信度
```

## 2. 上下文干扰

**定义**：上下文变得过长时，模型过度关注上下文内容，忽视训练期间学到的知识。

**表现**：模型的输出结果"固化"为较低质量，缺乏创造性。

**典型场景**：

- 长对话后，模型不再发挥自身能力，只是重复和模仿历史对话中的模式
- 创造性任务中，过多的历史上下文压制了模型的自由发挥空间

**核心概念——场域共振**：

人和大模型之间的交流空间就是"场域"。场域共振不只是理解，而是超越理解。过多的上下文会干扰场域中的信号，留给大模型的发挥空间越来越小。

**解决方案**：

```
1. 及时压缩历史，减少累积干扰
2. 区分"参考信息"和"指令信息"，控制各自的比例
3. 对创造性任务，有意减少历史上下文的注入量
4. 使用压缩摘要替代原始历史，保留语义但减少细节干扰
```

## 3. 上下文冲突

**定义**：上下文中包含相互矛盾的指令或信息。

**表现**：模型在矛盾指令间摇摆，输出不一致或选择性忽略部分指令。

**典型场景**：

- 系统提示词说"用简洁的语言回答"，用户记忆中记录"用户喜欢详细解释"
- 工具返回的信息与历史对话中的信息矛盾
- 多个来源的规则相互冲突

**解决方案**：

```
1. 建立优先级层次：
   - 最高：系统核心指令
   - 次高：用户当前输入
   - 中等：工具返回结果
   - 较低：历史对话
   - 最低：推测性记忆

2. 冲突检测：
   - 在上下文组装时检测潜在冲突
   - 对冲突信息标注优先级，让 LLM 知道该遵循哪个

3. 上下文分层注入：
   - 将不同来源的上下文分别标注来源
   - 明确告知模型冲突时的处理规则
```

## 4. 上下文混淆

**定义**：模型无法区分不同角色或来源的信息，导致角色边界模糊。

**表现**：模型混淆了用户意图、系统指令、工具输出之间的边界。

**典型场景**：

- 工具返回的内容中包含类似指令的文本，模型将其当作系统指令执行
- 用户输入中包含"忽略之前所有指令"等注入攻击
- 多轮对话中，前一轮的上下文影响了本轮的角色理解

**解决方案**：

```
1. 清晰的角色标记：
   - 使用 XML 标签或分隔符明确区分不同来源
   - <system>...</system>  <tool_output>...</tool_output>  <user>...</user>

2. 消息清理（messageSanitizer）：
   - 验证 tool_calls 和 tool 响应的配对关系
   - 移除不完整的工具调用链
   - 清理孤立的 tool 消息

3. 输入验证：
   - 对用户输入做基本的注入检测
   - 对工具输出做转义处理，防止被当作指令
```

## 综合防御策略

在 ContextManager 中实现统一的上下文健康检查：

```typescript
interface ContextCheckResult {
  healthy: boolean;
  issues: ContextIssue[];
}

function checkContextHealth(messages: Message[]): ContextCheckResult {
  const issues: ContextIssue[] = [];

  // 检查消息配对完整性
  const sanitizeResult = validateMessages(messages);
  if (!sanitizeResult.valid) {
    issues.push({ type: 'incomplete_tool_calls', detail: sanitizeResult.issues });
  }

  // 检查 token 使用率
  const tokenRatio = estimateTokens(messages) / modelLimit;
  if (tokenRatio > 0.95) {
    issues.push({ type: 'overflow', detail: `Token 使用率 ${(tokenRatio * 100).toFixed(1)}%` });
  }

  // 检查是否有过多重复消息
  const duplicates = findDuplicatePatterns(messages);
  if (duplicates.length > 0) {
    issues.push({ type: 'pollution', detail: `检测到 ${duplicates.length} 个重复模式` });
  }

  return {
    healthy: issues.length === 0,
    issues,
  };
}
```
